什么是 AI Agent — 一个生活化的故事

假设你想去旅行。你能找到 3 种"助手"：

助手类型能力对应技术
① 旅游手册 只能查信息，问什么答什么。问完了不会主动做事。 普通 LLM（如 ChatGPT 网页版）
② 智能客服脚本 有固定流程：选目的地→查机票→订酒店。每一步问你"是否继续"。 传统对话机器人 / 工作流
③ 私人助理 🌟 你说"帮我订下周去东京的便宜机票"，TA 自己查机票、比价、订票、发邮件给你。会自己决定下一步做什么。 AI Agent
🧠 一句话定义
Agent = LLM（大脑）+ 工具（双手）+ 循环（思考多步）

1.2 ReAct 范式 — Agent 怎么思考的

ReAct = Reasoning + Acting（思考 + 行动）。Agent 不是一次给出答案，而是反复这个循环：

用户问：「我下周去北京，要带什么？」

╭─ 第 1 轮 ─────────────────────────────────╮
│ 💭 Thought（思考）：                       │
│    我需要先知道北京下周的天气              │
│                                            │
│ ⚙️ Action（行动）：                        │
│    调用 get_weather(city="北京", days=7)  │
│                                            │
│ 👀 Observation（观察）：                   │
│    返回：周一晴 18°C，周二雨 15°C...      │
╰────────────────────────────────────────────╯
                  ↓
╭─ 第 2 轮 ─────────────────────────────────╮
│ 💭 Thought：温度 15-18°C 偏凉，          │
│    且有雨，需要薄外套+雨具                │
│                                            │
│ ⚙️ Action：无需更多工具，直接回答         │
│                                            │
│ 💬 最终回答：「建议带薄外套、雨伞、       │
│    保暖内衣。具体每天搭配..」              │
╰────────────────────────────────────────────╯

03

工具调用 — 给 LLM 装上"双手"（45 分钟）

3.1 为什么 LLM 需要工具

纯 LLM 有三大短板：

  • 🧠 知识有截止时间：Claude 不知道今天的新闻
  • 🔢 不擅长精确计算：让它算 3719 × 8261 经常出错
  • 🔒 无法操作外部世界：发不了邮件、查不了数据库

解决方案：让 LLM 学会调用工具。

3.2 最小例子：让 AI 算 2+2

新建 02_calculator.py：

from anthropic import Anthropic
from dotenv import load_dotenv

load_dotenv()
client = Anthropic()

# 1) 定义工具函数（真正干活的 Python 代码）
def add(a: int, b: int) -> int:
    return a + b

# 2) 告诉 LLM 有这个工具（JSON Schema 格式）
tools = [
    {
        "name": "add",
        "description": "计算两个整数的和",
        "input_schema": {
            "type": "object",
            "properties": {
                "a": {"type": "integer", "description": "第一个加数"},
                "b": {"type": "integer", "description": "第二个加数"}
            },
            "required": ["a", "b"]
        }
    }
]

# 3) 发起对话
response = client.messages.create(
    model="claude-sonnet-4-5",
    max_tokens=500,
    tools=tools,                                # 关键：传入 tools
    messages=[{"role": "user", "content": "请帮我算 137 + 246"}]
)

# 4) 看看 LLM 怎么回应
print("stop_reason:", response.stop_reason)
for block in response.content:
    print(f"  type={block.type}, content={block}")

运行后输出（关键部分）：

stop_reason: tool_use
  type=tool_use, content=ToolUseBlock(
    id='toolu_xxx',
    name='add',
    input={'a': 137, 'b': 246}
  )
🤯 这里发生了什么
LLM 看到工具描述后，自己决定这个问题需要调用 add，并自动从用户输入解析出 a=137, b=246。它没有给你最终答案 —— 它在等你"真正"执行这个函数。

3.3 完整流程：执行工具 + 把结果给 LLM

from anthropic import Anthropic
from dotenv import load_dotenv

load_dotenv()
client = Anthropic()

def add(a, b): return a + b

tools = [{
    "name": "add",
    "description": "计算两个整数的和",
    "input_schema": {
        "type": "object",
        "properties": {
            "a": {"type": "integer"},
            "b": {"type": "integer"}
        },
        "required": ["a", "b"]
    }
}]

def solve(question):
    messages = [{"role": "user", "content": question}]

    # —— 第 1 轮：LLM 决定调用工具 ——
    r1 = client.messages.create(
        model="claude-sonnet-4-5",
        max_tokens=500, tools=tools, messages=messages
    )
    messages.append({"role": "assistant", "content": r1.content})

    if r1.stop_reason != "tool_use":
        return r1.content[0].text  # LLM 不需要工具，直接回答

    # —— 执行工具 ——
    tool_block = next(b for b in r1.content if b.type == "tool_use")
    print(f"🔧 LLM 想调用 {tool_block.name}({tool_block.input})")
    result = add(**tool_block.input)
    print(f"📊 实际执行结果：{result}")

    # —— 第 2 轮：把工具结果交给 LLM，让它生成最终回答 ——
    messages.append({
        "role": "user",
        "content": [{
            "type": "tool_result",
            "tool_use_id": tool_block.id,
            "content": str(result)
        }]
    })

    r2 = client.messages.create(
        model="claude-sonnet-4-5",
        max_tokens=500, tools=tools, messages=messages
    )
    return r2.content[0].text

print(solve("帮我算 137 + 246"))

输出：

🔧 LLM 想调用 add({'a': 137, 'b': 246})
📊 实际执行结果：383
137 + 246 等于 383。

3.4 工具描述（schema）怎么写

schema 是给 LLM 看的"使用说明书"。写得越清楚，LLM 调用越准。

{
    "name": "search_books",                    # 工具名（蛇形）
    "description": "搜索图书馆中的书籍。"        # 关键！说清楚什么时候用
                   "适合需要查询书名、作者、"
                   "出版年份等信息时使用。",
    "input_schema": {
        "type": "object",
        "properties": {
            "keyword": {
                "type": "string",
                "description": "搜索关键词，如书名或作者"
            },
            "year_from": {
                "type": "integer",
                "description": "出版年份下限（可选）"
            }
        },
        "required": ["keyword"]               # 必填字段
    }
}
💡 写好 description 的 3 个原则
  1. What：这个工具是做什么的（1 句）
  2. When：什么场景下应该用（1-2 句）
  3. How：参数怎么填、返回什么格式
🎯 练习
给上面的代码再加一个工具 multiply(a, b)。问 LLM "(3+5) × 7 等于多少"，看它是否能自动调用两次工具（先 add，再 multiply）。
04

ReAct 循环 — Agent 的灵魂（60 分钟）

4.1 上一章的代码有个 bug

第 3 章的 solve() 只支持一次工具调用。如果问 "(3+5) × 7"，LLM 想先 add 再 multiply，但我们的代码只让它调 1 次就强制结束。

真正的 Agent 需要循环：调一次工具 → 看结果 → 决定再调还是结束 → 重复。

4.2 ReAct 循环的标准代码

新建 03_agent.py：

from anthropic import Anthropic
from dotenv import load_dotenv

load_dotenv()
client = Anthropic()

# ─── 工具实现 ───────────────────────────
def add(a, b): return a + b
def multiply(a, b): return a * b
def get_weather(city):
    fake = {"北京": "晴 22°C", "上海": "雨 18°C"}
    return fake.get(city, f"{city} 数据缺失")

TOOL_FUNCS = {
    "add": add,
    "multiply": multiply,
    "get_weather": get_weather,
}

# ─── 工具 Schema ────────────────────────
TOOLS = [
    {
        "name": "add",
        "description": "计算两个整数的和",
        "input_schema": {"type": "object",
            "properties": {"a": {"type":"integer"}, "b": {"type":"integer"}},
            "required": ["a","b"]}
    },
    {
        "name": "multiply",
        "description": "计算两个整数的乘积",
        "input_schema": {"type": "object",
            "properties": {"a": {"type":"integer"}, "b": {"type":"integer"}},
            "required": ["a","b"]}
    },
    {
        "name": "get_weather",
        "description": "查询指定城市的当前天气",
        "input_schema": {"type": "object",
            "properties": {"city": {"type":"string"}},
            "required": ["city"]}
    },
]

# ─── 核心：Agent 主循环 ───────────────────
def run_agent(user_message: str, max_steps: int = 10):
    """ReAct 循环：思考 → 调工具 → 观察 → 重复，直到完成或达上限"""
    messages = [{"role": "user", "content": user_message}]

    for step in range(1, max_steps + 1):
        print(f"\n━━━ Step {step} ━━━")

        # ① 让 LLM 思考下一步
        response = client.messages.create(
            model="claude-sonnet-4-5",
            max_tokens=2048,
            tools=TOOLS,
            messages=messages,
        )

        # ② 把 LLM 的整条回复（含 tool_use 块）加入历史
        messages.append({"role": "assistant", "content": response.content})

        # ③ 检查停止原因
        if response.stop_reason == "end_turn":
            # LLM 说完了，给出最终答案
            for block in response.content:
                if block.type == "text":
                    print(f"\n💬 最终答案：{block.text}")
            return

        if response.stop_reason != "tool_use":
            print(f"⚠️ 异常停止原因：{response.stop_reason}")
            return

        # ④ 执行所有工具调用
        tool_results = []
        for block in response.content:
            if block.type == "text" and block.text.strip():
                print(f"💭 思考：{block.text}")
            if block.type == "tool_use":
                print(f"🔧 调用 {block.name}({block.input})")
                func = TOOL_FUNCS.get(block.name)
                if not func:
                    result = f"错误：工具 {block.name} 不存在"
                else:
                    try:
                        result = func(**block.input)
                    except Exception as e:
                        result = f"工具执行出错：{e}"
                print(f"👀 结果：{result}")

                tool_results.append({
                    "type": "tool_result",
                    "tool_use_id": block.id,
                    "content": str(result),
                })

        # ⑤ 把所有工具结果作为 user 消息加入历史
        messages.append({"role": "user", "content": tool_results})

    print(f"\n⏱ 达到最大步数 {max_steps}，未完成。")


# ─── 测试 ────────────────────────────────
if __name__ == "__main__":
    run_agent("我要去北京出差，今天天气如何，建议穿什么？还有，(3+5)×7 等于多少？")

4.3 运行效果

━━━ Step 1 ━━━

4.4 必须理解的 5 个关键设计

设计为什么
max_steps 上限防止 LLM 死循环烧 token
把 assistant 整条回复加入 messages包含 tool_use 块的 ID，下一轮 LLM 才能匹配 tool_result
tool_result 包在 user 角色里API 规定：工具结果以 user 身份传回（counterintuitive 但必须）
循环里 try/except工具可能报错，错误信息要让 LLM 看到，它会自己重试或换方案
每轮把所有 tool_use 一起处理LLM 可能一轮请求多个工具，必须全部执行后一次性返回

  • 打印 response.usage 看 token 消耗，警惕过长上下文
  • 用 rich.print_json 漂亮打印 messages 调试
  • 给工具加详细日志，看 LLM 传的参数是否正确

5.4 工具 2：匹配度评估

def score_match(experience: str, keywords: list) -> dict:
    """评估一段经验描述与 JD 关键词的匹配度"""
    exp_lower = experience.lower()
    matched = [k for k in keywords if k.lower() in exp_lower]
    missing = [k for k in keywords if k not in matched]
    return {
        "matched": matched,
        "missing": missing[:5],                # 最缺的 5 个
        "ratio": round(len(matched) / max(len(keywords), 1), 2),
    }

# 测试

def rewrite_section(original: str, target_keywords: list, role: str) -> str:

1. 用 STAR 法则结构（背景-任务-行动-结果）

        model="claude-sonnet-4-5",     # 写作用强模型
        max_tokens=500,
        messages=[{"role": "user", "content": prompt}]
    )
    return r.content[0].text.strip()

import json
from anthropic import Anthropic
from dotenv import load_dotenv
from tools.jd_extractor import extract_jd_keywords
from tools.scoring import score_match
from tools.rewriter import rewrite_section

load_dotenv()
client = Anthropic()

TOOLS = [
    {"name": "extract_jd_keywords",
     "description": "从招聘 JD 中提取核心技术关键词",
     "input_schema": {"type":"object",
       "properties": {"jd": {"type":"string"}},
       "required": ["jd"]}},
    {"name": "score_match",
     "description": "评估一段简历经验与给定关键词的匹配度",
     "input_schema": {"type":"object",
       "properties": {
         "experience": {"type":"string"},
         "keywords": {"type":"array","items":{"type":"string"}}
       },
       "required": ["experience","keywords"]}},
    {"name": "rewrite_section",
     "description": "针对岗位关键词重写一段简历经验",
     "input_schema": {"type":"object",
       "properties": {
         "original": {"type":"string"},
         "target_keywords": {"type":"array","items":{"type":"string"}},
         "role": {"type":"string"}
       },
       "required": ["original","target_keywords","role"]}},
]

TOOL_FUNCS = {
    "extract_jd_keywords": extract_jd_keywords,
    "score_match": score_match,
    "rewrite_section": rewrite_section,
}

def run_resume_agent(resume: dict, jd: str, role: str):
    system = f"""你是简历优化顾问，帮用户优化简历投递"{role}"岗位。

工作流程：
1. 用 extract_jd_keywords 提取 JD 关键词
2. 用 score_match 评估每段经验匹配度
3. 对匹配度 < 0.5 的经验，用 rewrite_section 改写
{% raw %}4. 输出 JSON：{"rewritten": [...], "suggestions": [...]}{% endraw %}"""

    messages = [{"role":"user","content":
        f"简历：\n{json.dumps(resume, ensure_ascii=False, indent=2)}\n\nJD：\n{jd}"}]

    for step in range(1, 15):
        print(f"\n━━━ Step {step} ━━━")
        r = client.messages.create(
            model="claude-sonnet-4-5", max_tokens=4096,
            system=system, tools=TOOLS, messages=messages,
        )
        messages.append({"role":"assistant","content": r.content})

        if r.stop_reason != "tool_use":
            print("\n=== 最终输出 ===")
            for b in r.content:
                if b.type == "text": print(b.text)
            return

        tool_results = []
        for b in r.content:
            if b.type == "tool_use":
                print(f"🔧 {b.name}({list(b.input.keys())})")
                func = TOOL_FUNCS[b.name]
                try:
                    result = func(**b.input)
                except Exception as e:
                    result = f"错误：{e}"
                tool_results.append({"type":"tool_result",
                    "tool_use_id": b.id,
                    "content": json.dumps(result, ensure_ascii=False)})
        messages.append({"role":"user","content": tool_results})


if __name__ == "__main__":
    resume = {
        "experience": [{
            "title": "实习生",

要求：精通 Python，熟悉 Flask / FastAPI；
熟悉 Redis、MySQL；有大并发优化经验加分。"""

实战 ② RAG 个人知识库 Agent 4 小时 · 简历项目

6.1 RAG 是什么 — 一个直观的比喻

想象 LLM 是一个聪明但失忆的实习生。你问他"我们公司的报销流程是什么？"，他不知道（公司内部资料没在他的训练数据里）。

RAG (Retrieval-Augmented Generation) = 先让实习生"翻一下公司文件柜，找到相关页面，读完再回答"。

┌─ 准备阶段（一次性） ─────────────────────────────┐
│                                                   │
│  公司文档 ──► [切分成小块 chunks]                 │
│                       │                            │
│                       ▼                            │
│              [向量化 Embedding]                    │
│                       │                            │
│                       ▼                            │
│              [存入向量数据库]                       │
│                                                   │
└───────────────────────────────────────────────────┘

┌─ 查询阶段（每次问答） ──────────────────────────┐
│                                                   │
│  用户问题 ──► [向量化]                            │
│                  │                                 │
│                  ▼                                 │
│         [向量数据库相似度搜索]                     │
│                  │                                 │
│                  ▼                                 │
│         [取 Top-K 最相关 chunks]                  │
│                  │                                 │
│                  ▼                                 │
│         [拼到 prompt：你看这些内容回答]            │
│                  │                                 │
│                  ▼                                 │
│            [LLM 基于真实资料回答]                  │
│                                                   │
└───────────────────────────────────────────────────┘

6.2 关键概念

概念通俗解释
Embedding（向量化） 把一段文字变成 768 维数字数组。意思相近的文字，向量距离近。
向量数据库 专门存这些数字数组，能极快找出"和某个向量最像的 N 个"
Chunk（分片） 把长文档切成 200-500 字的小段，避免一次塞太多内容给 LLM
Top-K 检索 每次查询取最相关的 K 个 chunk（通常 K=3-5）

6.3 技术选型（推荐组合）

  • 向量数据库：ChromaDB（本地、零配置、免费）
  • Embedding 模型：bge-small-zh（中文优秀、本地可跑、免费）
  • LLM：Claude 或 DeepSeek

import chromadb
from sentence_transformers import SentenceTransformer
from pypdf import PdfReader
from anthropic import Anthropic
from dotenv import load_dotenv

load_dotenv()

# ─── 初始化 ────────────────────────────────────
embedder = SentenceTransformer("BAAI/bge-small-zh-v1.5")
chroma = chromadb.PersistentClient(path="./kb_storage")
collection = chroma.get_or_create_collection("my_notes")
llm = Anthropic()

# ─── 文档索引（运行一次即可）───────────────────
def chunk_text(text, size=500, overlap=50):
    """按字符切分，保留重叠避免上下文断裂"""
    chunks = []
    for i in range(0, len(text), size - overlap):
        chunks.append(text[i:i+size])
    return chunks

def index_pdf(path: str):
    reader = PdfReader(path)
    all_chunks = []
    for page_num, page in enumerate(reader.pages):
        text = page.extract_text() or ""
        for chunk in chunk_text(text):
            if chunk.strip():
                all_chunks.append(chunk)

    print(f"📄 解析 {len(reader.pages)} 页，生成 {len(all_chunks)} 个 chunk")

    # 批量向量化（一次性比逐个快）
    embeddings = embedder.encode(all_chunks, show_progress_bar=True).tolist()
    ids = [f"{path}-{i}" for i in range(len(all_chunks))]

    collection.add(
        documents=all_chunks,
        embeddings=embeddings,
        ids=ids,
        metadatas=[{"source": path} for _ in all_chunks]
    )
    print(f"✅ 已索引到向量数据库")

# ─── RAG 检索工具 ──────────────────────────────
def search_knowledge(query: str, top_k: int = 5) -> str:
    """从知识库检索与问题最相关的内容"""
    q_emb = embedder.encode([query]).tolist()
    results = collection.query(
        query_embeddings=q_emb,
        n_results=top_k
    )
    docs = results["documents"][0]
    if not docs:
        return "知识库中没找到相关内容"
    return "\n\n---\n\n".join([f"【片段 {i+1}】\n{d}" for i, d in enumerate(docs)])

# ─── Agent 主循环 ──────────────────────────────
TOOLS = [{
    "name": "search_knowledge",
    "description": "搜索用户个人知识库中与问题相关的资料。"
                   "在回答专业问题前必须先检索。",
    "input_schema": {
        "type": "object",
        "properties": {
            "query": {"type": "string", "description": "检索关键词或问题"},
            "top_k": {"type": "integer", "default": 5}
        },
        "required": ["query"]
    }
}]

SYSTEM = """你是用户的私人知识助手。

规则：
1. 回答专业问题前，必须先用 search_knowledge 检索
2. 严格基于检索到的内容回答，不要编造
3. 引用原文片段，让用户可追溯
4. 检索结果不相关时，明确告诉用户"知识库中暂无相关资料\""""

def ask(question: str):
    messages = [{"role": "user", "content": question}]

    for step in range(1, 6):
        r = llm.messages.create(
            model="claude-sonnet-4-5",
            max_tokens=2048,
            tools=TOOLS,
            system=SYSTEM,
            messages=messages,
        )
        messages.append({"role": "assistant", "content": r.content})

        if r.stop_reason != "tool_use":
            for b in r.content:
                if b.type == "text":
                    return b.text
            return ""

        results = []
        for b in r.content:
            if b.type == "tool_use":
                print(f"🔍 检索：{b.input['query']}")
                snippets = search_knowledge(**b.input)
                results.append({
                    "type": "tool_result",
                    "tool_use_id": b.id,
                    "content": snippets[:3000]
                })
        messages.append({"role": "user", "content": results})

# ─── 使用 ──────────────────────────────────────
if __name__ == "__main__":
    # 1) 首次：索引文档（运行一次后注释掉）
    index_pdf("我的考研笔记.pdf")

    # 2) 提问
    print(ask("线性代数中矩阵的秩有什么物理意义？"))

# ─── 浏览器工具 ─────────────────────────────────
async def open_url(url: str, page):
    await page.goto(url, wait_until="domcontentloaded", timeout=30000)
    return f"已打开 {url}，标题：{await page.title()}"

async def get_text(selector: str, page):
    """读取页面元素文本内容"""
    try:
        await page.wait_for_selector(selector, timeout=5000)
        el = await page.query_selector(selector)
        return (await el.inner_text())[:2000] if el else "未找到元素"
    except Exception as e:
        return f"错误：{e}"

async def click_element(selector: str, page):
    try:
        await page.click(selector, timeout=5000)
        return f"已点击 {selector}"
    except Exception as e:
        return f"点击失败：{e}"

async def screenshot(page, name="screenshot.png"):
    await page.screenshot(path=name, full_page=True)
    return f"已保存 {name}"

# ─── 工具 Schema ────────────────────────────────
TOOLS = [
    {"name":"open_url","description":"在浏览器中打开网页",
     "input_schema":{"type":"object",
       "properties":{"url":{"type":"string"}},"required":["url"]}},
    {"name":"get_text","description":"读取 CSS 选择器对应元素的文本",
     "input_schema":{"type":"object",
       "properties":{"selector":{"type":"string"}},"required":["selector"]}},
    {"name":"click_element","description":"点击 CSS 选择器对应元素",
     "input_schema":{"type":"object",
       "properties":{"selector":{"type":"string"}},"required":["selector"]}},
    {"name":"screenshot","description":"截全屏保存到文件",
     "input_schema":{"type":"object",
       "properties":{"name":{"type":"string"}}}},
]

# ─── Agent 主循环 ────────────────────────────────
async def browser_agent(goal: str):
    async with async_playwright() as pw:
        browser = await pw.chromium.launch(headless=False)
        page = await browser.new_page()

        messages = [{"role":"user","content": goal}]
        for step in range(1, 20):
            print(f"\n━━━ Step {step} ━━━")

            r = llm.messages.create(
                model="claude-sonnet-4-5", max_tokens=2048,
                tools=TOOLS, messages=messages,
                system="你能控制浏览器完成任务。每步先说要做什么，再调工具。"
            )
            messages.append({"role":"assistant","content": r.content})

            if r.stop_reason != "tool_use":
                for b in r.content:
                    if b.type == "text":
                        print(f"💬 {b.text}")
                break

            results = []
            for b in r.content:
                if b.type == "text" and b.text.strip():
                    print(f"💭 {b.text}")
                if b.type == "tool_use":
                    print(f"🔧 {b.name}({b.input})")
                    func_map = {
                        "open_url": open_url,
                        "get_text": get_text,
                        "click_element": click_element,
                        "screenshot": screenshot,
                    }
                    func = func_map[b.name]
                    out = await func(**b.input, page=page) if b.name != "screenshot" \
                          else await func(page=page, **b.input)
                    results.append({
                        "type":"tool_result",
                        "tool_use_id": b.id,
                        "content": str(out)[:2000]
                    })
            messages.append({"role":"user","content": results})

        await asyncio.sleep(2)
        await browser.close()

# ─── 跑起来 ──────────────────────────────────────
asyncio.run(browser_agent(
    "打开 https://www.zhipin.com，搜索 'AI Agent 工程师'，截屏第一页结果保存为 jobs.png"
))

MCP 协议入门（40 分钟）

8.1 MCP 是什么

MCP (Model Context Protocol) 是 Anthropic 推动的开放标准。类比：

  • USB-C：任何设备只要符合 USB-C 协议，就能互相充电、传数据
  • MCP：任何工具只要符合 MCP 协议，所有支持 MCP 的客户端（Claude Desktop / Claude Code / Cursor）都能直接调用

8.2 三方角色

┌──────────────┐     MCP 协议     ┌──────────────┐
│  MCP Client  │ ◄═══════════════►│  MCP Server  │
│ Claude Code  │                   │  你写的工具   │
│ Cursor       │                   │  (Python/TS) │
└──────────────┘                   └──────────────┘
        │
        ▼
  Claude/GPT 模型

8.3 写一个最小 MCP Server

pip install mcp
# weather_mcp.py
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("weather")

@mcp.tool()
def get_weather(city: str) -> str:
    """查询城市当前天气"""
    return f"{city} 今天 22°C 晴"

@mcp.tool()
def get_forecast(city: str, days: int = 3) -> list:
    """查询未来天气预报"""
    return [{"day": i+1, "city": city, "temp": 22+i} for i in range(days)]

if __name__ == "__main__":
    mcp.run()

FAQ & 避坑指南

Q1：Claude API 太贵 / 用不了，怎么办？
A：完全可以用 DeepSeek（国内、便宜）。代码改 2 行： 
from openai import OpenAI
client = OpenAI(
    api_key="sk-xxx",
    base_url="https://api.deepseek.com"
)
# 工具调用语法跟 OpenAI 完全一样
Q2：要不要学 LangChain？
A：建议先用原生 SDK 写一遍（理解原理），再学 LangChain（生产中常用）。 LangChain 把很多东西封装了，初学直接用反而不懂底层。
Q3：本地能跑大模型吗？
A：可以用 Ollama 跑 Llama 3 / Qwen 等开源模型。但工具调用稳定性不如 Claude/GPT， 学习阶段推荐用 API。
Q4：Agent 老是"瞎调工具"怎么办？
A：3 个调优方向：
  1. 工具 description 写得更具体，明确什么场景该用
  2. system prompt 加规则，如"必须先检索再回答"
  3. 限制工具数量，单次给 LLM 5 个以内最佳
Q5：怎么处理工具调用的错误？
A：把异常信息当作 tool_result 返回给 LLM，让它看到错误自己重试或换方案。不要 raise 出去中断循环。
Q6：上下文太长，token 超限怎么办？
A：
  • 每轮工具结果 truncate 到 N 字符
  • 历史消息保留最近 10 轮，前面的做摘要
  • 用 prompt caching（Anthropic 已支持，省 90% 成本）
Q7：项目时间不够，怎么取舍？
A：优先级 ① > ③ > ②。简历优化 Agent 最容易做且 HR 最有兴趣（你自己就是用户）； 浏览器自动化最炫酷适合展示；RAG 最有技术深度但耗时久。
Q8：学完后再看什么进阶资料？
A：
